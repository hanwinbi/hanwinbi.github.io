Hexo本地博客

我的第一个博客搭建与2016年，那时候才大一，也是跟着教程一起来的，三年过去了，早已经忘得不止东南西北了。考完试了，还是想把属于自己的小天地搭建起来，whatever，我的地盘我做主。

博客主题来源

以下搭建过程是我在进行本地预览的过程，为了避免以后重新设置的时候忘记，所以记录下来了。来源

安装Node.js

这个应为我在很早以前就配置好了环境，所以没换时间。写的过程中特地去搜了一下，一步到胃，咳咳，一步到位。
1.检查是否安装过homebrew，没有安装的安装命令如下
ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
2.安装node


检查是否安装完成npm -v、node -v
npm由于源服务器在国外下载node包经常超时，cnpm使用国内镜像，通过以下命令安装cnpm
npm install -g cnpm --registry=https://registry.npm.taobao.org
Nice done！

通过npm安装Hexo框架

1.前提是第一步已经完成了
npm install -g hexo-cli，可以用hexo -v查看版本号
2.创建一个空目录，cd到这个目录，然后要开始了！
输入hexo init初始化博客，用hexo new “BlogName”新建文章
3.生成静态文件
hexo g（g代表generate），生成博客静态文件
每次修改文章后，都需要通过hexo clean清理一下，
然后通过hexo g重新生成，
最后也不要忘了通过hexo s重新启动Hexo。
4.启动
hexo s，根据提示访问本地网页
5.更换主题
把下载的主题直接放在theme目录下，或者cd到theme目录，然后git clone也行啊。然后在_config.yml文件夹中Ctrl+F找到theme，换成你自己的主题的名字。
每次修改主题后，都需要通过hexo clean清理一下，
然后通过hexo g重新生成，
最后也不要忘了通过hexo s重新启动Hexo。

使用Gitment评论系统

开始
第一步 注册 OAuth Application

首先在点击这里注册自己OAuth Application，

此处有四个内容：

Application name： 随意填写undefinedHomepage URL: 随意填写一个urlundefinedApplication description: 随意填写undefinedAuthorization callback URL: 填你的博客网址（例: https://user.github.io/blog/），如果有域名则填域名。
注册完后你会得到一个 client ID 和一个 client secret，记住这两个玩意，等会hexo配置会用到

第二步 安装Gitment

在你的博客下使用npm进行安装gitment

$ npm install –save gitment
第三步 修改主题配置

因为这里我使用的是Next主题配置，在配置文件搜索关键字gitment,主要配置如下：

gitment:
enable: true //这里必须为开启
…
github_user: username
github_repo: blog_comments #新建一个存储评论的仓库，这里填写仓库名
client_id: #第一步注册的client_id
client_secret: #第一步注册的client_secret
…
github_user不知道是哪个？你在github页面中，点击你的头像看到第一行显示signed in as xxx，

就是填写这个xxx。github_repo这里是让你再新建一个仓库，用来存储评论的，不是当前的这个博客的仓库，

然后填上你仓库名的名字，对！就是单纯的名字，仓库名叫blog_comments就填blog_comments。
