## 云服务器配置
### 重置默认用户和 root 密码
因为之后有 SSH 连接此云服务器的需求，所以重置一下密码。
![[Pasted image 20220302145831.png]]
打开实例，在面板的右上角找到重置密码。用户名选项中有 root 和系统默认 Ubuntu 用户，两个密码都重新设置，并重启。腾讯云默认不允许 root 使用 SSH 远程登录，因此我们需要默认 Ubuntu 用户来辅助进入 root。
 
### 设置防火墙
在「网络信息」中找到防火墙「管理规则」，点击添加规则
![](https://github-ihs-1257225145.cos.ap-shanghai.myqcloud.com/picgo/Tencent_firewall_rules.png)
添加`22, 3389, 8080, 9900`和其他需要与客户端连接的开放端口号。

### 设置云服务器防火墙
在服务器面板我们可以直接点击一键登录服务器，或者使用本地的终端连接远程服务器（`ssh ubuntu@ip_addr`），`ip_addr` 需要使用网络信息中的「（公）」网地址。


```bash
# 切换到 root 用户
ubuntu@name_of_server:~$ su
# 输入在控制面板上重置的 root 用户密码，密码不会有回显
password:
# 打开防火墙
ubuntu@name_of_server:~$ sudo ufw enable
# 将端口号换成上图中的端口号，一个一个添加
ubuntu@name_of_server:~$ sudo ufw allow port
```
如果嫌通过 `IP` 地址登录比较麻烦，可以在腾讯云实例中找到「网站搭建指引」，购买域名。

## NPS
[NPS](https://ehang-io.github.io/nps/#/install)提供了较为详尽的配置方法，供查阅。

### 服务端配置（云服务器）
配置文件需要放在根目录（`/root`）下，所以我们切换到根目录，你可以放在根目录 下的任意位置。在 [ehang-io/nps](https://github.com/ehang-io/nps/releases) 找到对应版本的包，复制下载链接。

```bash
ubuntu@name_of_server:~$ cd /root
# 下载服务端程序
ubuntu@name_of_server:~$ wget https://github.com/ehang-io/nps/releases/download/v0.26.10/linux_amd64_server.tar.gz
# 解压文件
ubuntu@name_of_server:~$ tar -xzvf linux_amd64_server.tar.gz
```

编辑 `conf` 文件夹中的 `nps.conf` 文件，如下（百度 `vim` 的基本使用）。
```bash
appname = nps
#Boot mode(dev|pro)
runmode = pro

#HTTP(S) proxy port, no startup if empty
#http_proxy_ip=0.0.0.0
#http_proxy_port=80
#https_proxy_port=443
#https_just_proxy=true
#default https certificate setting
#https_default_cert_file=conf/server.pem
#https_default_key_file=conf/server.key

##bridge
bridge_type=tcp
bridge_port=9900 # 这个端口号需要打开，或者自己设置
bridge_ip=0.0.0.0

# Public password, which clients can use to connect to the server
# After the connection, the server will be able to open relevant ports and parse related domain names according to its own configuration file.
public_vkey=123

#Traffic data persistence interval(minute)
#Ignorance means no persistence
#flow_store_interval=1

# log level LevelEmergency->0  LevelAlert->1 LevelCritical->2 LevelError->3 LevelWarning->4 LevelNotice->5 LevelInformational->6 LevelDebug->7
log_level=7
#log_path=nps.log

#Whether to restrict IP access, true or false or ignore
#ip_limit=true

#p2p
#p2p_ip=
#p2p_port=

#web
web_host=a.o.com
web_username=admin # 这里是网页登录的用户名
web_password=123   # 网页登录的密码
web_port = 8080    # 网页的端口号，云服务器和实例上必须打开该端口号才能使用
web_ip=0.0.0.0
web_base_url=
web_open_ssl=false
web_cert_file=conf/server.pem
web_key_file=conf/server.key
# if web under proxy use sub path. like http://host/nps need this.
#web_base_url=/nps

#Web API unauthenticated IP address(the len of auth_crypt_key must be 16)
#Remove comments if needed
#auth_key=test
auth_crypt_key =1234567812345678

#allow_ports=9001-9009,10001,11000-12000

#Web management multi-user login
allow_user_login=false
allow_user_register=false
allow_user_change_username=false

#extension
allow_flow_limit=false
allow_rate_limit=false
allow_tunnel_num_limit=false
allow_local_proxy=false
allow_connection_num_limit=false
allow_multi_ip=false
system_info_display=true

#cache
http_cache=false
http_cache_length=100

#get origin ip
http_add_origin_header=false

#pprof debug options
#pprof_ip=0.0.0.0
#pprof_port=9999

#client disconnect timeout
disconnect_timeout=60
```

### 开启 NPS 服务
```bash
# 安装服务
ubuntu@name_of_server:~$ sudo nps install
# 启动服务
ubuntu@name_of_server:~$ sudo nps start
# 查看服务启动情况
ubuntu@name_of_server:~$ ps aux | grep nps
```
如果服务启动成功，可以打开浏览器，输入 `你的公网ip:8080` 就能看到登录界面了。
![](https://github-ihs-1257225145.cos.ap-shanghai.myqcloud.com/picgo/NPS.png)

### 客户端配置
参考[基本使用 (ehang-io.github.io)](https://ehang-io.github.io/nps/#/use)，写的比较清晰。

### 网页端配置
客户端 - 新增，只需要在「唯一验证密钥」中添加上客户端配置文件的密钥，保存即可。
![](https://github-ihs-1257225145.cos.ap-shanghai.myqcloud.com/picgo/nps_add_client.png)
就会出现以上的主机，点击隧道 - 新增，添加远程的服务端口号和目标，也就是需要内网渗透的主机地址。

返回主界面就可以看到「状态」和「连接」都在线了。
