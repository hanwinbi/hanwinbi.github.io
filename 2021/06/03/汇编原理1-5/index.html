    第一章 基础知识

    机器指令是一串二进制数，计算机通过改变高低电平，驱动电子器件，进行运算。汇编语言出现是因为0和1组成的机器码语言不便于阅读和编写。所以出现了汇编语言，它只是机器指令便于记忆的书写格式。

    代码执行大致是一个从 编程语言 -> 汇编语言 -> 机器指令 的过程。

    CPU中由 寄存器、ALU（数学运算单元）、 控制器和时钟四个部分组成。寄存器与内存的关系、ALU对寄存器中数据的操作是编译原理这门学科中主要探讨的内容。因为硬件设计和内部结构的差异，需要不同的电平脉冲来控制工作，所以每一种处理器有自己的指令集。这就是ARM架构和X86架构在最本质上的差异「具有不同的指令集」。

    寄存器，CPU中用于存储数据的器件。通过编译器将汇编指令转换成机器指令才能运行。

    汇编语言包括三种指令：

    • 汇编指令：机器码的更方便的书写方式
    • 伪指令：没有对应的机器码，编译器执行
    • 其他符号： + - * /，没有对应的机器码

    指令和数据在内存单元中的存在没有差别，都是二进制信息。1Byte是一个存储单元，存储单元从零开始顺序编号。

    CPU进行数据读写需要：

    • 存储单元的地址「地址线」
    • 操作类型，读、写「控制线」
    • 读或写的数据「数据线」


    专门用来连接CPU和其他芯片的导线叫做总线，总线有：地址总线、控制总线和数据总线。

    地址总线：一个CPU有N根地址线，说明这个CPU的地址宽度为N，寻址空间最大为2N2^N2N个内存单元，也就是说地址线的宽度限制了寻址空间（最大内存）
    数据总线：数据总线的宽度决定CPU和外界数据传送的速度。8根数据总线一次可传送一个8位二进制数据。
    控制总线：CPU对外部器件的控制通过控制总线进行的，控制总线的宽度决定CPU对外部器件的控制能力

    主板上的核心器件通过地址总线、数据总线和控制总线连接，器件包括CPU、存储器、外围芯片组、扩展卡槽等，扩展卡槽上一般有RAM内存条和各类接口卡。
    CPU和各类接口卡相连，间接的控制各种外设。

    存储器分为随机存储器和只读存储器

    • 随机存储器。主存储器=主板RAM+扩展卡槽RAM
    • 装有BIOS（Basic Input/Output System，基本输入输出系统）的ROM，是由主板厂商提供的软件系统
    • 接口卡上的RAM，显卡上的RAM

    虽然各类存储器在物理上是独立的，但是CPU都是通过总线相连，把他们都当作内存看待。不同的地址段控制不同的存储器。

    第二章 寄存器

    [[寄存器名字及其含义]]
    CPU由运算器、控制器、寄存器等器件组成，它们通内部总线相连，之前提到的数据总线、控制总线、地址总线相当于它们的外部总线。

    • 运算器信息处理
    • 寄存器存储数据
    • 控制器控制各种器件工作
    • 总线连接元件，进行数据传输

    不同CPU的寄存器数量不一样，8086CPU有14个存储器：AX、BX、CX、DX、SI、DI、SP、BP、IP、CS、SS、DS、ES、PSW

    AX、BX、CX、DX（16 bit）用来存放一般数据，称为通用寄存器，这四个寄存器可以分成两个8 bit的寄存器使用，
    AX = AH（8-15） + AL（0-7）（H-High L-Low）以此类推

    8086可以处理两种尺寸的数据

    • 字节 byte
    • 字 word：1 word = 2 byte

    mov: 赋值 mov ax, bx mov ax,18
    add:相加 汇编指令中不区分字母大小写，操作对象的位数需要保持一致 add ax,bx

    检测点 2.1

    1. F4A3H 31A3H 3123H 6246H
      826CH 6246H 826CH 04D8H
      0482H 6C82H D882H D888H
      D910H 3D36H

    16位结构的CPU

    • 运算器最多可以处理16位的数据
    • 寄存器的最大宽度为16 bit
    • 寄存器和运算器之间的通路为16 bit

    8086CPU内部通过两个16 位的地址合成的方法形成一个20位的物理地址

    • 两个16位地址，一个是段地址，一个是偏移地址
    • 地址加法器。物理地址 = 段地址x16+偏移地址
      • 物理地址 = 段地址x16+偏移地址的本质含义：CPU访问内存时，用一个基础地址（段地址x16）和一个相对于基础地址的偏移地址相加，给出物理地址。
      • 为什么是乘16？我把它思考成地址高位右移，右移一位就相当于除以16，那么一个16位的地址就能存储一部分20位的地址，得到真实物理地址的时候就需要乘以16

    段的概念
    并不是将内存分成了一个一个的段，每个段有一个段地址。❌
    内存并没有分段，段的划分来自于CPU，相当于将段地址作为一个基准，而内存中每一个物理地址都可以作为一个段地址来使用。

    检测点 2.2

    1. 0010H 1000FH
    2. 1000H 2000H

    段寄存器

    8086有四个段寄存器：CS、DS、SS、ES，访问内存时由这四个寄存器提供的内存单元段地址
    CS：代码段寄存器 IP：指令指针寄存器
    CS:IP指向内存地址，它指向的内容是指令的物理地址

    汇编指令更改CS、IP地址的指令：jmp CS:IP，可以单独修改IP的值，jmp ax，ax中放的是需要跳转到的指针地址

    检测点 2.3

    1）3次 IP+6

    常用的Debug功能（功能命令可以小写）

    • R 查看、改变CPU寄存器的内容
    • D 查看内存中的内容: d CS:IP
    • E 改写内存中的内容: e CS:IP data data

    • T 执行一条汇编指令
    • A 以汇编指令的格式在内存中写入一条机器指令

    第三章 寄存器（ 内存访问）

    一个字由两个连续的内存单元组成，高地址存放字型数据的高位，低地址存放字符数据的低位。

    一个字的表示：N地址字单元 = N内存单元+ （N+1）内存单元

    要访问内存中的数据，要用DS寄存器，DS：存放要访问的数据的段地址。
    将内存单元中的数据放入寄存器中。mov register_name,memory_unit_add，用[...] 表示一个内存单元，中间的数值表示内存单元的偏移地址，段地址存放在DS中。

    mov
    add
    sub
    分段的作用：把数据、代码的内容分开保存，读取的过程就知道读取的是指令还是数据

    检测点 3.1

    1. 2662 E626 E626 2662 D6E6 FD48 2C14 0000 00E6 0000 0026 000C

    栈

    入栈和出栈操作push pop
    通过CS:IP我们可以知道执行指令的未知，要知道栈的位置需要栈寄存器SS:SP。任意时刻SS:SP指向栈顶元素

    8086CPU中没有检测栈顶是否超出了栈空间，超出栈空间会覆盖超出部分原始的数据。所以需要自行控制程序不超过栈空间

    CS:IP指向代码段，SS:SP指向栈段，DS指向数据段

    入栈的过程：

    1. SP = SP - 2
    2. 高位放高位数据，低位放低位数据

    出栈的过程：

    1. 读取数据
    2. SP = SP + 2

    第四章 编写程序

    汇编语言源程序->编译连接
    可执行文件包括两个部分：

    • 程序（源程序中的汇编指令编译得到的机器码）+ 数据
    • 相关的描述信息（程序有多大，要占用多少内存空间）

    汇编语言源程序 = 伪指令 + 汇编指令
    编译器根据伪指令进行相关的编译工作

    • 一个段必须有一个名称来标识：
    • end 汇编程序的结束标记，和ends区别开
    • assume 假设某一段寄存器和程序中的某一个用segment...ends的段相关联，使用 assume将有特定用途的段和相关的段寄存器关联起来。

    一个程序结束后，将CPU的控制权交给使它得以运行的程序，这个过程为：程序返回

    编写一个源程序后缀为.asm，使用masm编译，使用link连接
    简化方式编译和连接masm 路径\文件名; ，文件类型可以是.asm 也可以是.txt编译过程中自动忽略中间文件的生成，link 路径\文件名;，生成可执行文件，忽略中间文件的生成。

    在DOS运行一个程序的时候，是由command将程序从可执行文件中加载入内存，command设置CPU的CS:IP指向程序的第一条指令，从而使程序运行，程序结束后返回command，CPU继续运行command。

    连接的作用：

    1. 当源程序很大时，可以将它分为多个源程序文件编译，每个源程序编译成为目标文件后，再用连接将他们连接到一起，生成一个可执行文件；
    2. 程序中调用了某个库文件中的子程序，需要将这个库文件和该程序生成的目标文件连接在一起，生成一个可执行文件
    3. 一个源程序编译后，得到了存有机器码的目标文件，目标文件中的有些内容还不能直接用来生成可执行文件，连接程序将这些内容处理为最终的可执行信息。

    程序段的前256个字节中创建一个程序段前缀（PSP）

    代码运行到int 21时使用P命令结束

    第五章 [BX]和 loop 指令

    [BX] 和 [0] 类似，只是偏移地址由原来的 [0] 现在存放在 BX 中。

    inc bx 的含义是 bx 中的内容加1

    Loop 指令：

    1. (cx) = (cx)-1;
    2. 判断 cx 中的值，不为零则转到标号处执行程序，如果为零则向下执行。
      注：()描述一个寄存器或一个内存单元中的内容

    通常我们会使用 loop 实现循环功能，cx 中存放循环次数
    要实现循环的三个要点：

    1. 在cx中存放循环的次数
    2. 开始循环的第一条指令前写上标号
    3. 结束位置（开始循环判断的位置）loop 标号

    在汇编语言中，标号代表一个地址，s: add ax,ax 在 s 这个地址处有一条指令 add ax,ax
    汇编源程序中，数据不能以字母开头，在字母前面要加上一个0

    debug name.exe 可以跟踪程序，进入debug 后可以使用 g IP 跳过之前的命令，跳转到loop指令前，使用p命令会自动重复执行loop中的指令

    debug 和 汇编程序对指令的不同处理

    mov ax,[0] 在 debug 中表示将 ds:0 处的数据送入 ax 中，汇编程序经过编译器会被当作指令 mov ax,0 处理。在汇编程序中所以需要用[bx]来访问内存单元，这时候段地址会默认在 ds 中或者使用ds:[num]显式的给出段地址所在的寄存器。

    显式的指明内存单元的段地址的ds:、cs:、ss:、es:，在汇编语言中成为 段前缀
